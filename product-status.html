Оновлено: 13.04.2026
ТЗ Tool
Перетворює будь-який пакет файлів від клієнта на структуроване ТЗ — зрозуміле виконавцю, ПМу і AI.
Принцип: ТЗ вважається готовим до виробництва, якщо після його читання можна почати моделінг без дзвінка клієнту.
ДЛЯ КОГО
Виконавець
3D Візуалізатор
Потребує повне ТЗ по стадіях: що моделювати, які матеріали, ракурси, дедлайн — без повторних запитань клієнту.
Менеджмент
ПМ / Арт Директор
Контролює повноту ТЗ, бачить що відсутнє, генерує запит до клієнта, має звіт по проекту.

PIPELINE — 8 ЕТАПІВ
01
Прийом і класифікація файлів
ГОТОВО
Сортування не за типом файлу, а за типом даних всередині. AI визначає категорію кожного файлу.
  • Завантаження PDF, DOCX, DWG, DXF, Excel, CSV, зображень, TXT
  • AI-класифікація файлів по категоріях (Планування, Стиль, Матеріали…)
  • AI-класифікація кожної сторінки PDF окремо
  • Галерея сторінок з вибором і ручною корекцією категорій
  • Покращений парсинг PDF: реконструкція тексту по рядках, PNG для сканів
  • Індикатор текстового шару — скільки сторінок мають текст vs скан
02
Визначення типу проекту і SOW-шаблон
ГОТОВО
AI визначає тип проекту та підвантажує еталон повного ТЗ для цього типу.
  • 5 типів проекту: Інтер'єр житловий / комерційний, Екстер'єр, Мастерплан, Продукт
  • SOW-шаблони з переліком обов'язкових пунктів для кожного типу
03
Валідація і детектор протиріч
ГОТОВО
Порівняння з SOW-шаблоном. Детектор конфліктів між файлами.
  • Порівняння з шаблоном: що є, чого не вистачає, що неповно
  • Детальний опис неясностей — що знайдено і що конкретно запитати
  • Детектор протиріч між файлами (стиль конфліктує, різні матеріали для однієї поверхні…)
  • UI-панель "Конфлікти" в перегляді ТЗ із джерелами та питаннями для уточнення
04
Запит недостатнього + дефолти по типу проекту
ЧАСТКОВО
Формування client request. Дефолтні значення для незаповнених пунктів.
  • ✓ Кнопка "Запит" — генерує готовий текст з питаннями для клієнта
  • ✗ Дефолти по типу проекту ("Не вказано час доби → дефолт: золота година, 17:00")
  • ✗ Клієнт підтверджує дефолти або надсилає заміну
05
Парсинг по виробничих стадіях
ГОТОВО
Кожна вимога прив'язана до стадії виробництва і містить цитату з джерела.
  • 6 стадій: Моделінг, Текстуринг, Світло, Камери, Пост-продакшн, Видача
  • Перегляд ТЗ за стадіями (поруч із переглядом за кімнатами)
  • Поле quote — дослівна цитата з брифу до кожної вимоги
06
Нормалізація в єдиний формат
ЧАСТКОВО
Все вхідне конвертується в єдиний читабельний документ.
  • ✓ Копія ТЗ в Markdown (кнопка MD) — базова версія, лише текст по кімнатах
  • ✗ Повний .md файл з усіма вимогами, зображеннями і посиланнями на джерела
07
Фінальна структура ТЗ
ЧАСТКОВО
10-розділовий документ: від опису проекту до пріоритетів виробництва.
  • ✓ Структурований перегляд за кімнатами і категоріями
  • ✓ Перегляд за виробничими стадіями
  • ✓ Анотація проекту (повний текст)
  • ✓ Прив'язка вимог до зображень-джерел із превью
  • ✓ Редагування і видалення вимог вручну
  • ✓ Друк в PDF
  • ✗ Розділ "Пріоритети" — що важливіше якщо потрібно скоротити
08
Звіт для ПМа і клієнта
НЕ РОЗПОЧАТО
Внутрішній звіт для команди і зовнішній документ для клієнта.
  • Таблиця: пункт ТЗ / статус / джерело / стадія
  • Список протиріч з описом
  • Зовнішній документ для клієнта — що прийнято, що дефолт, що критично відсутнє

AI КЛАСИФІКАЦІЯ
Авто-сортування файлів
Claude Haiku класифікує кожен файл і кожну сторінку PDF по категоріях проекту.
ВИРОБНИЧІ СТАДІЇ
ТЗ по стадіях
Кожна вимога прив'язана до стадії: Моделінг → Текстуринг → Світло → Камери → Пост → Видача.
ЦИТАТИ
Дослівно з брифу
Кожна вимога містить цитату з джерела — що саме сказав клієнт, без інтерпретацій.
SOW ВАЛІДАЦІЯ
Детектор прогалин
Порівнює ТЗ з шаблоном проекту. Показує що відсутнє і що неповно з деталями. Детектує протиріччя між файлами.
ЗАПИТ КЛІЄНТУ
Готовий текст
Кнопка "Запит" генерує структурований список питань для відправки клієнту.
ДЖЕРЕЛА
Прив'язка до файлів
Кожна вимога посилається на файл і сторінку-джерело. Превью відкривається прямо в ТЗ.
СЕСІЇ
Збереження прогресу
ТЗ зберігається локально. При наступному відкритті можна продовжити з того ж місця.
РОАДМАП
План виробництва
AI будує впорядкований план задач по стадіях з урахуванням залежностей між ними.
